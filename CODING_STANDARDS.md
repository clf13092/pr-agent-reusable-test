# プロジェクト固有のコーディング標準

## Python コーディング規約

### 必須要件
- PEP 8に準拠すること
- 関数名はsnake_caseを使用
- クラス名はPascalCaseを使用
- 定数は大文字のSNAKE_CASEを使用

### セキュリティ要件
- ハードコーディングされた機密情報は禁止
- APIキー、パスワード、トークンは環境変数を使用
- SQLインジェクション対策を実装

### フォーマット要件
- インデントは4スペース
- 行の最大長は88文字
- 関数間は2行空ける
- import文は標準ライブラリ、サードパーティ、ローカルの順

## Terraform コーディング規約

### 命名規則
- リソース名は小文字とハイフンを使用
- 変数名はsnake_caseを使用
- モジュール名は小文字とアンダースコアを使用

### セキュリティ要件
- セキュリティグループは最小権限の原則
- IAMポリシーは必要最小限の権限
- パブリックアクセスは明示的に許可された場合のみ

### フォーマット要件
- terraform fmtを実行済みであること
- 適切なインデントを使用
- リソース間は1行空ける
